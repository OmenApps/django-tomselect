Constant Filter By Demo

This example demonstrates filtering with constant values that are always applied:

  • The Published Articles field always filters to show only articles with status=published
  • This constant filter is applied regardless of user interaction
  • You can optionally select a Magazine to further narrow down the results
  • Constant filters are useful for enforcing business rules (e.g., only show active items)
Code Example
from django_tomselect.app_settings import TomSelectConfig, Const

# Mixed filters - field-based AND constant
published_articles = TomSelectModelMultipleChoiceField(
    config=TomSelectConfig(
        url="autocomplete-article",
        filter_by=[
            ("magazine", "magazine_id"),       # Optional magazine filter
            Const("published", "status"),      # Always filter to published
        ],
    ),
)

How It Works

Constant filters use the Const("value", "lookup") helper to always include a filter parameter. The URL will include f='__const__status=published' which the server interprets as a constant filter (not dependent on a form field). This is useful for enforcing business rules like "only show active/published items" without requiring a visible form field.

Use Cases
  • Content Management: Only show published articles/pages
  • E-commerce: Only show active/in-stock products
  • User Management: Only show active users
  • Multi-tenant: Always filter by current organization
